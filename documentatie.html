Introducere

Baza de date

Vom opta pentru a folosi MySQL, datorita faptului ca este sub o licenta deschisa, iar faptul ca este usor de interogat a reprezentat un plus.

Tabelul Users

Datele ce vor fi introduse in acest tabel sunt furnizate de form-ul din pagina de register si corespund credentialelor contului nou creat.

Asa arata tabelul users, ca structura, acesta fiind tabelul unde vor fi stocate toate detaliile legate de un utilizator:

  • id
  • password
  • email
  • admin(yes/no)

Tabelul Services

Datele ce vor fi introduse in acest tabel sunt furnizate de form-ul din pagina de admin.

Asa arata structura tabelului recipes:

  • category
  • city
  • price
  • start_date
  • end_date

Front End

Toate paginile site-ului sunt responsive, elementele ce le compun redimensionandu-se in pagina in functie de dispozitivul utilizat.

Register+Login

Inregistrarea si logarea se vor face pe pagina login.html.

Parteile de register si login sunt intuitive. Pentru a completa trebuiesc introduse urmatoarele date: e-mail, parola. Completarea tuturor acestor campuri este obligatorie.

Input box-ul destinat introducerii parolei va avea ca tip "password pentru ca textul introdus inauntru sa fie cenzurat.

Campul pentru parola este cenzurat pentru a proteja datele utilizatorului

Nu se va putea apasa butonul "Submit" fara ca toate campurile sa fie completate folosindu-se formatul specificat.

Logout

Pentru deconectarea utilizatorul logat trebuie sa apese butonul de log-out, ce se afla in dreapta sus in navbar, pe orice pagina.

Servicii

Pagina de servicii este populata cu diferite oferte de servicii.

La un serviciu se afiseaza urmatoarele detalii: numele serviciului, valabilitate si pret.

Adauga servicii

Pe pagina de admin, pentru un utilizator logat cu grad de admin.

Pagina de admin contine un formular cu toate campurile obligatorii. Campurile sunt: categorie, pret, oras, valabilitate.

Dupa ce formularul este completat, avem un buton "Add service" de tip submit.

Back End

Register & Login

Arhitectura generala a partii/modulului responsabil de Register si Login o sa fie bazata pe MVC.

Fiecare utilizator inregistrat va avea un email si un id unic.

Va fi o functionalitate de a schimba din register in login si vice-versa.

Register

Pentru a ne creea un cont se verifica daca datele transmise POST sunt corecte.

Dupa ce se realizeaza, unui cont nou creat i se vor salva campurile transmise in baza de date, dupa modelul user, iar parola contului nou creat este salvata.

Login

Pentru a realiza logarea utilizatorului vom trimite un request la back-end cu metoda POST. Dupa validarea datelor si verificarea userului in baza de date, daca acesta exista.

Servicii

La incarcarea paginii de shop, se vor afisa serviciile. In realizarea functionalitatii se va utiliza metoda GET.

Pe pagina de verificam daca userul este logat.

Adaugare serviciu

Formularul de adaugare serviciu trebuie trimis cu toate campurile completate.

Shop

Serviciul este luat din baza de date printr-un request de forma GET.

Toate informatiile primite despre un servicu vor fi afisate: categorie, oras, pret, valabilitate.
